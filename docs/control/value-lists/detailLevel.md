# detailLevel
**Scope Note:**
The completeness or exhaustiveness of the description. Archivists may prescribe locally defined elements to satisfy each level of detail.

- SAA dictionary. [https://dictionary.archivists.org/entry/level-of-description.html](https://dictionary.archivists.org/entry/level-of-description.html)

---

## basic
**Scope Note:**
Extends the minimal detail level and adds further recommended elements, e.g. based on local description conventions.

- DACS single-level optimum: [https://saa-ts-dacs.github.io/dacs/06_part_I/02_chapter_01.html#single-level-optimum](https://saa-ts-dacs.github.io/dacs/06_part_I/02_chapter_01.html#single-level-optimum)
- MARC21 17-Encoding level for minimal level bibliographic description: [https://www.loc.gov/marc/bibliographic/bdleader.html](https://www.loc.gov/marc/bibliographic/bdleader.html)

## extended
**Scope Note:**
Extends to further description elements creating added value for certain application contexts, e.g. URIs for Linked Data.

## minimal
**Scope Note:**
Uses only the mandatory elements as per the schema definition of the EAS and covers the essential elements of the related ICA description standards.

 - DACS single-level required: [https://saa-ts-dacs.github.io/dacs/06_part_I/02_chapter_01.html#single-level-required](https://saa-ts-dacs.github.io/dacs/06_part_I/02_chapter_01.html#single-level-required)
